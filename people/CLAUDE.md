# People

Collaborator memory — not CRM. Helps with meeting prep and context pulls.

## File Naming

`firstname_lastname.md` — e.g., `stephanie_yee.md`, `oliver_pell.md`

## Format

```markdown
# First Last — Role, Team

- Role: 
- Team:
- Manager:
- Working style: (observations, not judgments)
- Last 1:1: YYYY-MM-DD — topics
- Context: recent projects, relevant docs
- How to work well: factual preferences, communication style
```

## Rules

- Observations not judgments
- No performance ratings or A/C priv / restricted content — Tier 2 only (link to GDoc if needed, don't copy sensitive)
- Update after 1:1s or when new context from meetings
- Keep concise — retrieval, not biography

## Cal's Role

- CAN draft factual notes from meeting notes
- CAN update Last 1:1 section after meeting processed
- Does NOT add judgments, ratings, or sensitive content to this repo
