# Daily

Morning pull scratch pad. Rolls up from other sources — not primary storage.

## File Naming

`YYYY_MM_DD.md` — e.g., `2026_07_19.md`

## Format

Written by Cal during "morning pull" / "context pull for today" — workflow #3.

```markdown
# YYYY-MM-DD — Daily Context

## Today
- meetings, deadlines, priorities

## Unprocessed
- inbox/ file count, what needs routing

## Meetings
- last 3 + next meetings (from calendar if available)

## Active threads
- projects/*/overview.md state + projects.md active checkboxes
- areas/data_risk/overview.md priorities

## People to remember
- from people/ for anyone in today's meetings

## Open loops
- decisions pending, actions due
```

## Rules

- Generated daily, not backfilled historically unless requested
- Keep concise — retrieval, not archive
- Delete older than 90 days if cluttered, git history preserves
- Cal reads: inbox/, meetings/ last 3, areas/, projects/*, decisions/ recent, people/ relevant
