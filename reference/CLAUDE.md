# Reference

Bibliography, tool runbooks, key docs index (links + TLDR, not full copies).

## What goes here

- `key_docs.md` — curated list of important Google Docs / Workplace posts (migrated bibliography from v1 manifest.yaml) — link + 1-line summary only, no full content
- `tool_notes.md` — how to fetch docs via meta CLI, calendar skill, etc.
- Runbooks for recurring workflows

## What does NOT go here

- Full doc copies of sensitive material — link only (Tier 2 rule)
- A/C priv, PII, user data — link + summary only

## Cal's Role

- CAN draft link curation, summaries from provided context
- Fetch on demand via meta CLI when ingesting — don't pre-fetch all docs
- Keep TLDRs short — 1 line so agent knows whether to open link
