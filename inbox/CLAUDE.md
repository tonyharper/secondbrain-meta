# Inbox

Quick capture. Inbox zero goal — process daily.

## Structure

- `meetings/` — raw transcripts, notes, Google Doc pastes. File naming `YYYY_MM_DD_{source}_{topic}.md` or `{date}_{slug}.txt`. Keep original verbatim at top, add `Source:` header if applicable.
- `docs/` — URLs or raw doc drops to ingest into team brains. Naming `{date}_{slug}.md` with first line `Source: <URL>` if from Drive/Workplace/Docs.
- `notes/` — quick thoughts, ideas that need routing.

## Workflow

1. Drop file here (manual or via /capture later)
2. Tony says "process meeting X" or "ingest doc X"
3. Cal reads file + relevant area/project CLAUDE.md
4. Cal routes to proper place (meetings/, projects/, areas/, reference/)
5. Cal deletes/moves original from inbox/

## Rules

- Inbox is not storage. If file sits > 7 days, it's stale — delete or route.
- No subfolders beyond the three.
- Keep raw verbatim — don't summarize before routing.
